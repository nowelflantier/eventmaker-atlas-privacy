Privacy Policy

Eventmaker Atlas Overlay · Last updated: April 23, 2026

Overview

Eventmaker Atlas Overlay is a Chrome extension that provides a read-only configuration documentation overlay for the Eventmaker back-office. This policy explains what data the extension accesses, how it is used, how it is stored, and what it does not do.

Data accessed

The extension may access the following data within app.eventmaker.io:

  • Eventmaker configuration data — the extension makes read-only GET requests to the Eventmaker API to retrieve configuration objects such as guest fields, segments, forms, pages, categories, email templates, and automations.
  • Configuration data only — the extension is designed to process configuration and structural data only, not individual participant profile data or participant exports.
  • Current page URL — the extension reads the URL of the active Eventmaker page to detect context (which event, which section) and display relevant information.
  • Extension state — the extension stores lightweight local state such as UI preferences, navigation state, and cache metadata.

Authentication

No usernames, passwords, or authentication credentials are collected, stored, or transmitted by the extension. It relies solely on the authenticated Eventmaker session already active in your browser.

Data storage

All accessed data is stored locally on your device only:

  • IndexedDB — used to cache Eventmaker configuration snapshots locally, so the extension can work without repeated network calls.
  • chrome.storage.local — used to store lightweight extension state such as UI preferences, navigation state, and cache-related data.

None of this data leaves your device. No accessed data is sent to the developer’s servers. It is never transmitted to any third party or analytics service.

Data we do not collect

  • We do not collect or sell user data for advertising or marketing purposes.
  • We do not use analytics or tracking tools.
  • We do not share any data with third parties.
  • We do not transmit accessed data to any server other than the Eventmaker endpoints already used in your authenticated session.

How data is used

  • To analyze configuration dependencies inside Eventmaker.
  • To display contextual documentation directly in the Eventmaker back-office.
  • To detect configuration anomalies such as broken references or invalid Liquid syntax.
  • To improve performance through local caching only.

Permissions used

  • storage — to persist local cache and extension state.
  • Host permission for https://app.eventmaker.io/* — to inject the overlay into Eventmaker back-office pages and make read-only requests to the Eventmaker API required for the extension’s functionality.

Data deletion

Locally cached data can be removed by clearing the extension’s local browser storage or by uninstalling the extension.

Contact

If you have any questions about this policy, please contact the extension maintainer within your organization.

Politique de confidentialité

Eventmaker Atlas Overlay · Dernière mise à jour : 23 avril 2026

Présentation

Eventmaker Atlas Overlay est une extension Chrome qui affiche une surcouche de documentation de configuration en lecture seule dans le back-office Eventmaker. Cette page explique quelles données l’extension peut accéder, comment elles sont utilisées, comment elles sont stockées, et ce qu’elle ne fait pas.

Données accédées

L’extension peut accéder aux données suivantes sur app.eventmaker.io :

  • Données de configuration Eventmaker — l’extension effectue des requêtes GET en lecture seule vers l’API Eventmaker pour récupérer des objets de configuration comme les champs invités, segments, formulaires, pages, catégories, templates email et automatisations.
  • Données de configuration uniquement — l’extension est conçue pour traiter des données de structure et de configuration, et non des données individuelles de participants ni des exports de participants.
  • URL de la page courante — l’extension lit l’URL de la page Eventmaker active pour détecter le contexte (quel événement, quelle section) et afficher les informations pertinentes.
  • État de l’extension — l’extension stocke localement des données légères comme les préférences d’interface, l’état de navigation et des métadonnées de cache.

Authentification

Aucun identifiant, mot de passe ou credential d’authentification n’est collecté, stocké ou transmis par l’extension. Elle s’appuie uniquement sur la session Eventmaker déjà authentifiée dans votre navigateur.

Stockage des données

Toutes les données accédées sont stockées localement sur votre appareil uniquement :

  • IndexedDB — utilisé pour mettre en cache localement les snapshots de configuration Eventmaker, afin d’éviter des appels réseau répétés.
  • chrome.storage.local — utilisé pour stocker des données légères d’état de l’extension, comme les préférences d’interface, l’état de navigation et certaines données liées au cache.

Aucune de ces données ne quitte votre appareil. Aucune donnée accédée n’est envoyée vers les serveurs du développeur. Elles ne sont jamais transmises à un tiers ni à un service d’analyse.

Données non collectées

  • Nous ne collectons ni ne vendons de données utilisateur à des fins publicitaires ou marketing.
  • Nous n’utilisons aucun outil d’analyse ou de tracking.
  • Nous ne partageons aucune donnée avec des tiers.
  • Nous ne transmettons aucune donnée accédée vers un serveur autre que les endpoints Eventmaker déjà utilisés dans votre session authentifiée.

Utilisation des données

  • Analyser les dépendances de configuration dans Eventmaker.
  • Afficher une documentation contextuelle directement dans le back-office Eventmaker.
  • Détecter des anomalies de configuration, comme des références cassées ou une syntaxe Liquid invalide.
  • Améliorer les performances uniquement via un cache local.

Permissions utilisées

  • storage — pour persister le cache local et l’état de l’extension.
  • Permission hôte pour https://app.eventmaker.io/* — pour injecter la surcouche dans les pages du back-office Eventmaker et effectuer les requêtes en lecture seule vers l’API Eventmaker nécessaires au fonctionnement de l’extension.

Suppression des données

Les données mises en cache localement peuvent être supprimées en vidant le stockage local de l’extension dans le navigateur ou en désinstallant l’extension.

Contact

Pour toute question relative à cette politique, contactez le mainteneur de l’extension au sein de votre organisation.
